# Job Stack Analyzer

Десктопное приложение для анализа вакансий с **Remotive.com**.
Показывает среднюю зарплату, топ навыков с процентами и активных работодателей.

> **Почему Remotive, а не hh.ru?**
> hh.ru требует специальный User-Agent и может блокировать запросы по IP.
> Remotive — полностью открытый публичный API, без ключей, без регистрации, всегда работает.

## Установка и запуск

### 1. Установить зависимости
```bash
pip install -r requirements.txt
```

### 2. Настроить PostgreSQL (необязательно)
Создать базу данных:
```sql
CREATE DATABASE job_analyzer;
```
При запуске приложение покажет диалог настройки БД.
Если PostgreSQL нет — нажмите «Пропустить», приложение работает и без него.

### 3. Запустить приложение
```bash
python main.py
```

## Как работает анализ навыков

Remotive API не возвращает навыки отдельным полем (в отличие от hh.ru).
Поэтому мы сами анализируем текст описания вакансии:

```python
for keyword in TECH_KEYWORDS:           # берём слово из нашего словаря
    pattern = r'\b' + keyword + r'\b'   # ищем его как отдельное слово
    if re.search(pattern, text_lower):  # если нашли...
        found_skills.append(keyword)    # добавляем в список
```

Потом для каждого навыка считаем процент:
```python
percentage = (count / total_vacancies) * 100
```

## Шпаргалка для защиты

### Вопрос 1: «Почему Remotive, а не hh.ru?»

**Ответ:** Мы сначала использовали hh.ru, но он стал возвращать 403 (Forbidden) — это значит,
что сервер отклонил наш запрос. Возможные причины: слишком много запросов с одного IP,
или сервер посчитал нас ботом. Remotive — открытый публичный API специально для разработчиков,
никогда не блокирует, работает без ключей и регистрации.

---

### Вопрос 2: «Почему не используете key_skills, а парсите текст?»

**Ответ:** Remotive API не даёт навыки отдельным полем — только описание вакансии в HTML.
Мы убираем HTML-теги через `re.sub(r'<[^>]+>', ' ', text)` и ищем технологии из
заранее составленного словаря TECH_KEYWORDS. Это честнее, чем ничего не показывать.

---

### Вопрос 3: «Как считаются проценты навыков?»

**Ответ:** Для каждой вакансии мы ищем навыки в тексте. Потом считаем, в скольких вакансиях
встретился каждый навык. Делим на общее число и умножаем на 100.

Пример: Python нашли в 40 из 50 вакансий → 40 / 50 × 100 = 80%.

```python
percentage = (count / total_vacancies) * 100
```

---

### Вопрос 4: «Почему средняя зарплата считается только по тем, где она указана?»

**Ответ:** Если вакансия без зарплаты — мы не знаем реальную сумму.
Учитывать её как 0 занизит среднее. Правильнее — считать только по тем, где цифры есть.
В коде: если `salary_str` пустой — делаем `continue` (пропускаем).

---

### Вопрос 5: «Почему зарплата в долларах, а не рублях?»

**Ответ:** Remotive — международная платформа для удалённой работы, там зарплаты в USD.
Это честно отражено в интерфейсе. Если нужны русские вакансии — можно подключить
другой API, например SuperJob (у него есть открытый API).

---

### Вопрос 6: «Зачем нужна PostgreSQL?»

**Ответ:** Для истории поисков. При каждом успешном поиске мы сохраняем запрос,
дату, количество вакансий и топ-навыки. При следующем запуске история загружается из БД.
Это показывает умение работать с реальной базой данных через psycopg2.

---

### Вопрос 7: «Почему UI не замерзает во время загрузки?»

**Ответ:** Запрос к API и анализ запускаются в `QThread` — это Qt-механизм для фоновых задач.
Пока поток работает, главный поток обслуживает интерфейс. Когда поток заканчивает —
отправляет сигнал `finished`, и UI обновляется. Это стандартный способ в PyQt6.
